\begin{itemize}
    \item Selecting a method is only the first step in ML projects
    \item Users often need support to start implementation
    \item Templates and reusable workflows can reduce effort
    \item Such support can lower the barrier for non-experts
\end{itemize}

\section{Semantic Retrieval and Text Similarity}
\begin{itemize}
    \item Text can be represented as embeddings (vectors)
    \item Similarity measures can compare meaning between texts
    \item Used to match problem descriptions with relevant content
    \item More flexible than keyword-based search
\end{itemize}

\section{Literature-Based Discovery / Retrieval}
\begin{itemize}
    \item Scientific articles are a rich but unstructured knowledge source
    \item Challenges include large volume, noise, and unclear relevance
    \item Methods can be linked to articles through extraction and metadata
    \item Semantic retrieval can improve matching between problems and literature
\end{itemize}

\section{Human-in-the-Loop and Feedback}
\begin{itemize}
    \item User input helps guide and refine recommendations
    \item Feedback can improve system performance over time
    \item Requires large user base and careful design to avoid bias
\end{itemize}

\section{Web-Based ML Tools / Interfaces}
\begin{itemize}
    \item User interfaces allow interaction with the system
    \item Should support clear input and understandable output
    \item Transparency helps users trust recommendations
    \item Focus is on usability, not detailed UX design
\end{itemize}

\section{ML in Product Development and Production}
\begin{itemize}
    \item Go through findings from the literature review in pre-master project
    \item ML is used for tasks such as Quality prediction and defect detection, Process optimization, Predictive maintenance, Time series forecasting
    \item Product development often has limited and uncertain data
    \item Production systems may have structured and continuous data
